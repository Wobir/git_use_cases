# создание SSH подключения
SSH - предпостительный протокол для работы с удаленными репозиториями поэтому нужно уметь создавать ключи и помещать на удаленный репозиторий  
git Bash(win 10 и win 11) или Терминал на Linux: __ssh-keygen__  
если старая весрия windows(почему-то): необходима программа __Putty__.  
при использовании ssh-keygen в папке пользователя появляется скрытая папка __.ssh/__ в ней будут находится файлы ключей. они создаются парами различаются расширением __.pub__.  
Файл с таким расширением мы можем передавать безопасно на нужные сервера/пк. Открываем файл блокнотом видим примерно такую запись: __"ssh-ed25519 AAAAC3NzaC1l1NTE5AAAAIt0AxO2uFif0F42RYBr+9HnXJpBgCJ9T teacher_wobir@ws-r-teacher"__ - Это публичная часть ключа её нужно скопировать и поместить в настройках аккаунта в блок **SSH and GPG keys**

# Клонирование репозитория 
После того как вы сделали удаленный репозиторий его можно склонировать на свой компьютер и потом отправлять в него коммиты всего лишь тремя командами.  
Для клонирования репозитория переходим на основную панель и нажимаем зеленую кнопку __code__ далее переключаемся на SSH(настроили его выше) и копируем ссылку.  
Дальше открываем git Bash (терминал на Linux) и пишем команду  
__git clone <сслыка на ssh репозитория>__  
После нажатия enter в папке пользователя появится локальная копия репозитория, мы можем помещать файлы в эту папку и потом отправлять на удаленный репозиторий.  

# Отправка на удаленный репозиторий
__git add .__  - Добавить ВСЕ изменения, ИСКЛЮЧАЯ то что указано в файле .gitignore,  в следующий коммит  
__git commit -m "Название коммита"__ - Сформировать коммит, если репозиторий только локальный этого достаточно для сохранения версии репозитория  
__git push__ - Отправка коммита на удаленный репозиторий, если репозиторий
